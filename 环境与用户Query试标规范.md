# 环境 & 用户 Query 试标规范

（试标通过的数据可以用作正式数据正常结算）

---

## 本次试标范围

本次试标只构建仓库 Dockerfile 和 5-20 条 prompt，后续正式作业会涉及模型调用。

---

## 1. 任务描述

本任务主要围绕真实代码仓库构建可复现环境，并产出一批贴近真实用户在 Cursor、Trea 等 Agent 产品中提出的自然语言请求。

### 1.1 核心目标

标注的核心目标有两个：

1. **编写初始环境的 Dockerfile**：为每个仓库准备一个可在虚拟机中复现的初始环境
2. **基于该仓库产出 5~20 条真实、可执行、具有一定多样性的用户任务**（基于原始仓库内容可改造程度，不可以硬编）

### 1.2 标注原则

标注时应遵循以下原则：

- 任务应尽量贴近真实开发场景，而非考试题或抽象问答题
- 任务应建立在当前仓库真实代码上下文之上
- 任务应该是在 docker 中能执行的
- 任务应能驱动 Agent 围绕当前仓库开展具体工作，包括但不限于修复、迭代、重构、测试、分析、工程化处理，以及基于仓库上下文的代码理解、知识问答和问题解释等
- 同一仓库下的任务应避免高度重复，需体现一定多样性

---

## 2. 标注流程

### 2.1 仓库选择

标注人员先选择一个适合标注的仓库，并创建一条父记录。

> 公有仓库和私有仓库都需要，一条公有一条私有，交错标注

仓库应满足以下条件：

- **必须是日常开发使用的**
- 具备真实开发上下文
- 可以在 docker 中构建和复现
- 代码结构足以支撑至少 5 条任务
- 不建议选择过于空泛、纯 demo、过小或无法运行的仓库

### 2.2 准备环境

围绕该仓库准备环境，包括：

- Dockerfile
- repo 文件夹

环境准备要求如下：

1. Dockerfile 中必须设置 `WORKDIR /app`
2. 容器内仓库目录统一为 `/app`
3. 必须确保容器内仓库状态为 Agent 处理任务前的初始现场
4. 若当前仓库快照不是目标现场，应在 Dockerfile 中直接 checkout 到正确现场
5. 所有项目都必须完成 Git 初始化
6. Dockerfile 需完成最小可用初始化，包括安装必要依赖、创建工作目录、预留 app 目录，并提供默认 shell 或默认命令（可选）

### 2.3 验证环境

需自检环境是否满足以下条件：

1. 镜像可成功构建
2. 容器可正常启动
3. 当前工作目录为 `/app`
4. `repo` 存在（不涉及可留空）
5. `repo` 为 Git 仓库（不涉及可留空）
6. 容器内代码为任务起始现场
7. 进入容器后无需额外手工初始化即可开始工作

### 2.4 Prompt 任务

围绕该仓库设计 5~20 条任务（基于原始仓库设计难度），并创建对应子记录。

#### 任务设计要求

- **Prompt 必须要确保在给定的 docker 环境中是可运行的问题**（我们现在不支持 DinD）
- 尽量模拟真实用户在 Agent 中的表达方式
- 应建立在当前仓库真实代码上下文之上，题目中最好涉及/引用到原始代码片段，保证分布足够真实

#### 任务类型

- 可包含需要修改代码的任务
- 也可包含不需要修改代码的理解、分析、解释或问答类任务
- 问答类任务必须与当前仓库、代码实现、模块设计、运行现象或工程配置直接相关，不应设计为脱离仓库上下文的泛知识问答

#### 任务多样性要求

- 同一仓库下的任务不得高度重复
- 应覆盖不同难度、不同类型或不同模块，体现一定多样性

## 3. 字段描述

| 字段名 | 必填 | 说明 | 表头层级 |
|--------|------|------|----------|
| repo_url | 是 | 仓库地址或来源信息 | 父记录 |
| repo_type | 是 | 仓库属性：私有仓库/公有仓库 | 父记录 |
| language | 是 | 仓库主要是用的编程语言 | 父记录 |
| dockerfile | 是 | 上传的 dockerfile 附件 | 父记录 |
| repo | 是 | repo中项目的初始文件；用压缩包提交 | 父记录 |
| environment_notes | 否 | 环境补充说明 | 父记录 |
| task_count | 是 | 当前仓库下任务数量，要求 5~20 条 | 父记录 |
| task_index | 是 | 任务序号 | 子记录 |
| prompt | 是 | 真实用户在 Agent 中输入的原始请求 | 子记录 |
| difficulty | 是 | 任务难度，按附录评分规则标注相应难度分级 | 子记录 |
| category | 是 | 任务类别，按附录类型表选择对应任务类型 | 子记录 |
| tech_stack | 是 | 技术栈标签，按照需求产物填写；多个值用逗号分隔 | 子记录 |
| module_tags | 是 | 功能模块标签，按照需求产物填写；多个值用逗号分隔 | 子记录 |

## 附录 C：Dockerfile 示例

### C.1 Linux dockerfile

```dockerfile
# 基础镜像 (自定义，可选择一个最合适当前项目的 base image)
FROM ubuntu:22.04

# 设置工作目录
WORKDIR /app

# 把宿主机当前目录下 xxx 仓库文件夹内的"所有内容"复制到容器的 /app 目录下
COPY xxx/ ./

# (可选) 安装必要环境 等操作
RUN apt-get update && apt-get install -y --no-install-recommends \
    tmux \
    git \
    curl \
    python3 \
    python3-pip \
    && rm -rf /var/lib/apt/lists/*  # 使用 pip3 安装 Python 依赖
RUN pip3 install --no-cache-dir pandas scipy numpy scikit-learn
```
